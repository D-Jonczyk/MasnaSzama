Use Case: Zamawianie jedzenia
Language: Polish
Preconditions:
  • Klient musi podac miasto po ktorym beda wyszukiwane restauracje.
Postconditions:
  • Zamawianie jedzenia z wybranej restauracji.
Actors:
  • Klient
Main flow:
  1. System wyswietla pole do wpisania miasta, aby mogl wyswietlic restauracje.
  2. Klient wpisuje nazwe miasta.
  3. System sprawdza czy aplikacja obsluguje Miasto.
  4. System wyswietla liste restauracji.
  5. Klient wybiera restauracj.
  6. System wyswietla informacje o restauracji oraz wszystkie dania z cena.
  7. Klient wybiera danie i dodaje je do zamowienia.
  8. Klient wybiera opcje zaplac.
  9. System wyswietla metody platnosci.
  10. Klient wybiera opcje platnosci oraz wprowadza potrzebne dane.
  11. System dokonuje walidacji formularza platnosci.
  12. System wyswietla prosbe o adres na ktory ma zostac dostarczone jedzenie.
  13. Klient wprowadza adres i zatwierdza.
  14. System sprawdza czy wszystkie dane zostaly podane poprawnie.
  15. System przenosi klienta na strone startowa.
Alternate flow:
  • 3 Nie ma takiego miasta GOTO 1
  • 11 Podano zle dane sprobuj jeszcze raz. GOTO 9
  • 14 Podano zly adres, sprobuj ponownie. GOTO 12

Use Case: Zapamietywanie miasta.
Language: Polish
Preconditions:
  • Uzytkownik musi byc zalogowany.
Postconditions:
  • Zapamietywanie miasta zalogowanego uzytkownika.
Actors:
  • Klient
Main flow:
  1. Klient wybiera opcje dodaj miasto.
  2. System wyswietla formularz do wpisania miasta.
  3. Klient wpisuje miasto, ktore chce aby system zapamietal jako domyslne.
  4. System dokonuje walidacji formularza.
  5. System ustawia wybrane miasto jako domyslne.
  6. System powraca do panelu uzytkownika.
Alternate flow:
  • 4 Nie ma takiego miasta na naszej liscie. GOTO 2
