Use Case: Dodanie restauracji do systemu
Language: Polish
Preconditions:
  • Aktor jest zalogowany
  • Aktor posiada konto restauratora
Postconditions:
  • Wysłanie prośby o dodanie restauracji do systemu
Actors:
  • Właściciel restauracji
Main flow:
  1. Aktor otwiera podstronę "Panel restauratora"
  2. Aktor naciska przycisk "Dodaj restauracje"
  3. System wyświetla formularz zgłoszeniowy
  4. Aktor wypełnia formularz danymi dotyczącymi restauracji
  5. Aktor naciska przycisk "Zatwierdź"
  6. System wyświetla prośbę o dokumenty potwierdzające, że aktor jest właścicielem restauracji
  7. Aktor naciska przycisk "Wyślij zgłoszenie"
  8. System wysyła formularz do administratora serwisu
  9. Aktor otrzymuje powiadomienie o procesie weryfikacji dodawania restauracji do systemu
Alternate flow:
  • 4 Błędne dane GOTO 3
  • 9 Odrzucono zgłoszenie. Błędne dane GOTO 3
  • 9 Odrzucono zgłoszenie. Brak wystarczających dokumentów potwierdzających posiadanie restauracji GOTO 6

Use Case: Modyfikowanie karty dań do restauracji
Language: Polish
Preconditions:
  • Aktor jest zalogowany
  • Aktor posiada konto restauratora
Postconditions:
Actors:
  • Właściciel restauracji
Main flow:
  1. Aktor otwiera podstronę "Panel restauratora"
  2. System wyświetla wszystkie restauracje, którymi obecnie zarządza aktor
  3. Aktor naciska przycisk przyporządkowany danej restauracji
  4. System wyświetla panel zarządzania danej restauracji
  5. Aktor naciska przycisk "Modyfikuj kartę dań"
  6. System wyświetla kartę dań
  7. Aktor naciska przycisk przy daniu, które chcę zmodyfikować
  8. System wyświetla formularz
  9. Aktor modyfikuje dane dotyczące posiłku
  10. Aktor naciska przycisk "Zatwierdź"
  11. System zapisuję zmodyfikowaną pozycję do bazy danych
Alternate flow:
  • 7 Aktor naciska naciska przycisk "Dodaj nowy element" GOTO 8
  • 7 Aktor naciska przycisk "Usuń" przy daniu, które chce usunąć GOTO 10
  • 10 Błędne dane GOTO 9

Use Case: Modyfikowanie stanu zamówienia
Language: Polish
Preconditions:
  • Aktor jest zalogowany
  • Aktor posiada konto pracownicze
Postconditions:
Actors:
  • Pracownik restauracji
Main flow:
  1. Aktor otwiera podstronę "Panel restauracji"
  2. System wyświetla wszystkie zamówienia dotyczące danej restauracji wraz z opisem i statusem płatności
  3. Aktor naciska przycisk "Potwierdź" przy zamówieniu, które obecnie realizuje
  4. System zmienia status zamówienia na "przyjęte"
  5. Aktor po skompletowaniu zamówienia naciska przycisk "Zrealizowano"
  6. System zmienia status zamówienia na "zrealizowano"
  7. System wysyła informację do kuriera o zleceniu
Alternate flow:
  • 3 Aktor naciska przycisk "Anuluj zamówienie", gdy stwierdza że zamówienie nie może zostać wykonane GOTO 2
